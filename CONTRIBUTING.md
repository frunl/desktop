# Contributing To Frunl Desktop

Thanks for helping improve Frunl Desktop. This public repository is for downloads, release notes, bug reports, and feature requests.

## Before Opening An Issue

1. Check the [latest release](https://github.com/frunl/desktop/releases/latest) and confirm you are using the newest available build.
2. Search existing [open and closed issues](https://github.com/frunl/desktop/issues) for the same problem or request.
3. Remove passwords, API keys, access tokens, customer data, and private workspace details from screenshots, logs, and reproduction notes.

## Bug Reports

Use the [bug report form](https://github.com/frunl/desktop/issues/new?template=bug_report.yml) for crashes, install problems, sign-in problems, broken workflows, confusing behavior, or anything that appears to be a defect.

Helpful bug reports include:

- The Frunl Desktop version.
- The macOS version.
- Whether your Mac is Apple Silicon or Intel.
- The exact steps that reproduce the issue.
- What you expected to happen.
- What actually happened.
- Redacted logs or screenshots when they make the issue clearer.

## Feature Requests

Use the [feature request form](https://github.com/frunl/desktop/issues/new?template=feature_request.yml) for product ideas, workflow improvements, missing integrations, and quality-of-life requests.

Helpful feature requests include:

- The job or workflow you are trying to complete.
- Why the current product is not enough.
- Any workaround you use today.
- The outcome you would expect from the improvement.

## Sensitive Reports

Do not open a public issue for private account data, credentials, customer data, or privacy requests.

For account-specific support, contact support@frunl.com.

For privacy requests or sensitive data concerns, contact privacy@frunl.com.

## Pull Requests

This public repository does not accept direct code contributions.

If you want to help, open a clear issue first. A Frunl maintainer will review the report or request and decide how to handle it.
